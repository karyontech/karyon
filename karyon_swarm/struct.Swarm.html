Expand description

Swarm layer on top of Node. Manages protocol-aware peer groups and advertises swarm membership in the Node’s bloom filter.

Each call to join registers a new swarm; a single Swarm instance can manage many concurrent swarms across different protocols (and optionally different instances per protocol). By default a swarm’s SwarmKey is derived from the protocol id alone; use Swarm::join_with_instance for sub-grouping inside a single protocol (e.g. chat rooms, pub/sub topics).

Joining a swarm also adds the swarm key to the Node’s optional bloom, so other peers can discover this node via Node::find_peers_with / Swarm::find_peers without paying the cost of a full handshake first.
