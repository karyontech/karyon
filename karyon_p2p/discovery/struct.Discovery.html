This method will attempt to connect to a peer in the routing table. If the routing table is empty, it will start the seeding process for finding new peers.

This will perform a backoff to prevent getting stuck in the loop if the seeding process couldnโt find any peers.

async fn start_seeding(&self)

Starts seeding process.

This method randomly selects a peer from the routing table and attempts to connect to that peer for the initial lookup. If the routing table doesnโt have an available entry, it will connect to one of the provided bootstrap endpoints in the Config and initiate the lookup.
